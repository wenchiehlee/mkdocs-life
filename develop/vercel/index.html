    如何將 github pages 遷移到 vercel 上託管

    前言

    早期網站使用 github pages,後來遷移到 coding,最近又放到騰訊雲網站靜態託管,無論是 coding 的 cos 儲存桶,還是靜態網站託管

    他們都是收費的,那有沒有免費的託管商呢,既不影響網站的訪問速度還免費,於是,找了一下,還真有,vercel和Netlify,就是免費的

    其中大名頂頂的Next.js,create-react-app,Nuxt.js等就是部署在部署託管在vercel的,而vuejs,reactjs等就是託管在Netlify上的

    想必經常開啟這些鼎鼎大名官網的時候,訪問速度還是可以的

    今天說一下這個vercel,Vercel 可以部署任何前端應用程式的最佳場所。號稱以零配置部署到我們的全球邊緣網路,動態擴充套件到數百萬個頁面而不會費力

    一鍵匯入(支援 github,gitlab,BitBucket),自己的儲存庫，然後推送。vercel 內建的CI / CD系統會在每次程式碼更改時觸發

    體驗過後,確實方便,強大

    01

    為什麼選擇 vercel

    ⒈ 免費部署託管前端應用

    ⒉ 支援一鍵匯入(github,gitlab),零配置

    ⒊ 支援自定義域名以及配置 ssl 證書,https.

    ⒋ 簡單友好,個人賬戶免費,可以部署 next.js,Nuxt.js,Gatsby.js,Angular,Ember.js,Hexo,Eleventy,Hugo,Docusaurus2,Preact,Dojo,Vue.js,Jekyll 等

    02

    前提條件

    ⒈ 先成功部署 github pages(本文以 github pages 為例,其他類似的應用也是如此)

    ⒉ 然後登陸 vercel 官網, 一鍵匯入

    03

    怎麼使用呢

    在自己的 github 倉庫中找到 github pages 倉庫

    (找到自己github pages的倉庫)

    (在vercel上匯入github 倉庫的程式碼)

    等待匯入部署即可

    04

    自定義域名解析

    點選 Settings?Domains? 輸入自己的域名，並在域名購買方控制檯(在阿里買的域名就去阿里控制檯,在騰訊雲買的域名,就去騰訊雲的控制檯)新增域名解析（CNAME）

    解析完成後即可透過自己的域名訪問自己的部落格了

    05

    總結

    當你不滿足於 github pages,嫌棄它訪問得慢,是可以選擇 vercel 來進行託管的,也支援自定義域名,免費的一個 ssl 證書

    只要一鍵匯入程式碼就可以了的,非常簡單方便,可以一鍵部署前端很多應用..
